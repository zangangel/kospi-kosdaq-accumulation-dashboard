# KOSPI/KOSDAQ 매집비 대시보드

한국 주식(코스피/코스닥)의 증권사별 매수·매도 데이터를 CSV로 넣어 **매집비(A1/A2/A3)** 를 계산하고 종목별로 비교하는 정적 웹 대시보드입니다.

> 주의: 이 프로젝트는 공개 영상 설명에서 확인 가능한 기준을 바탕으로 만든 **투명한 근사 모델**입니다. 비공개/유료/특허성 로직을 복제하지 않으며, 매수·매도 추천이 아닙니다.

## 핵심 기능

- CSV 업로드 또는 샘플 데이터 로드
- 네이버증권 거래원정보 기반 관심종목 자동 업데이트
- 당일/5일/20일/60일 누적 기간 전환
- 종목별 A1/A2/A3 점수 및 등급 계산
- 시가총액 구간별 해석 포인트 자동 표시
- 최근 창 vs 장기 창 비교를 위한 기간 필터
- 거래원/증권사 집중도, 순매수 상위 브로커 표시
- 모든 로직이 `src/accumulation.js`에 공개되어 검토 가능

## 빠른 실행

```bash
npm test
npm run fetch:naver
npm run serve
```

브라우저에서 `http://localhost:5173` 접속.

## 자동 업데이트

`data/watchlist.csv`에 관심종목을 넣으면 GitHub Actions가 평일 한국시간 16:30에 네이버증권 거래원정보를 수집해 `data/auto/latest.json`을 갱신합니다. 사이트는 이 자동 데이터를 우선 표시하고, 기간 선택에서 당일/5일/20일/60일 누적 매집비를 전환할 수 있습니다.

자세한 내용은 [`docs/auto-update.md`](docs/auto-update.md)를 참고하세요.

## CSV 형식

```csv
date,ticker,name,market,broker,buy,sell,net_buy,float_shares,listed_shares,market_cap_krw,close
2026-06-01,005930,삼성전자,KOSPI,미래에셋,120000,50000,70000,5969782550,5969782550,480000000000000,80000
```

- `net_buy`가 비어 있으면 `buy - sell`로 계산합니다.
- `float_shares`가 없으면 `listed_shares`를 보수적 대체값으로 사용합니다.
- 같은 종목·기간·브로커 행을 합산해 계산합니다.

## 점수 공식(공개 설명 기반 근사)

| 항목 | 의미 | 근사 공식 | 등급 |
|---|---|---|---|
| A1 | 대형주/주도주 매수 강도 | `총매수 / 총매도 * 100` | A ≥ 300, B ≥ 200, C ≥ 130, D < 130 |
| A2 | 전체 거래원 순매집 비율 | `양수 순매수 합 / 유통주식수 * 100` | A ≥ 5%, B ≥ 3%, C ≥ 1.5%, D < 1.5% |
| A3 | 상위 3개 거래원 집중 매집 비율 | `상위3 양수 순매수 / 유통주식수 * 100` | A ≥ 5%, B ≥ 3%, C ≥ 1.5%, D < 1.5% |

## 해석 가이드

- 대형주: 첫 글자(A1)를 더 중요하게 봅니다.
- 1조~10조 중형주: A1과 A2를 함께 봅니다.
- 5천억 이하/저유통 소형주: A3 집중도를 더 중요하게 봅니다.
- 장기 창이 좋아도 최근 창이 DDD면 최근 매수 지속성이 약해졌을 수 있습니다.

## 한계

- 거래원 원자료는 사용자가 HTS/MTS/데이터벤더에서 CSV로 제공해야 합니다.
- 비공개 매집 알고리즘, 세력 추정, 특허 로직은 재현하지 않습니다.
- 가격 추세, 거래대금, 공시/뉴스, 락업, 유동성을 함께 확인해야 합니다.
